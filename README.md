Это общая часть dock.
### Функции
dock на хосте умеет:
```
dock list
dock makeimg
dock clean
dock upgrade [id]
dock term [id]
dock kill [id]
dock exec [id] [cmd]
dock dexec [id] [cmd]
dock getno [id]
dock getname [id]
dock getip [id]
```
Где `id` может быть номер экземпляра (целиком либо первые цифры), либо путь к астре для которой был запущен экземпляр
Если на хосте только один экземляр с нужным префиксом, то id необязательно

`ins` - запустит контейнер окружения и откроет терминал

`list` - перечислит все образы и контейнеры принадлежащие dock

`makeimg` - пересоберёт docker образ оружения

`clean` - удалит все контейнеры принадлежащие entrance-dock

`upgrade [id]` - обновить образ из выбранного контейнера

`term [id]` - откроет терминал окружения выбранного экземпляра

`kill [id]` - удалить выбранный экземпляр

`exec [id] [cmd]` - запустить команду внутри контейнера в интерактивном режиме

`dexec [id] [cmd]` - запустить команду внутри контейнера в detached режиме

`getip [id]` - получить ip адрес экземпляра

`getno [id]` - получить номер экземпляра

`getname [id]` - получить номер экземпляра


## Разработка
Если приходится часто пересобирать образ, то можно использовать apt-cacher-ng, это кеширующий прокси для пакетов убунты. На хосте
```
sudo apt install apt-cacher-ng
```
Все автоматически поднимется, а dock его подхватит во время выполнения следующиего apt install. Посмотреть что закешировано можно в /var/cache/apt-cacher-ng
