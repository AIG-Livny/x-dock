# x-dock
Утилита для создания стандартизированного Docker-окружения под конкретное приложение. Позволяет быстро развернуть среду разработки и подключиться к ней через SSH или VS Code Remote.

Зачем нужно Docker-окружение для разработчика?
- Воспроизводимость: при смене ОС или компьютера рабочее окружение восстанавливается одной командой.
- Единообразие: все пути к файлам и зависимости фиксированы и заранее известны.
- Готовность к работе: всё необходимое настроено «из коробки».
- Изоляция: среда разработки не засоряет хостовую систему.
- Синхронизация: вся команда использует идентичные настройки независимо от конфигурации личных ПК.

# Инструкция

## Создание своего my-app-dock
Эта часть является ядром и предполагается, что будет включаться в каждый my-app-dock как submodule
Вместо x в имени "x-dock" подставляется имя вашего приложения.

- Создайте папку проекта и инициализируйте репозиторий: `git init`.
- Добавьте ядро как субмодуль: `git submodule add https://github.com/AIG-Livny/x-dock.git common`
- Скопируйте структуру файлов из папки example.
- Настройте конфигурацию под свои задачи.
- Выполните ./dock install. После этого в системе появится глобальная команда my-app-dock.

Полный список функций доступен по команде my-app-dock help.

Список функций выводится командой `my-app-dock help`

## Первое использование

Перед началом работы необходимо подготовить образ:

- Соберите образ командой my-app-dock makeimg (или получите готовый у коллег).
- Перейдите в папку с основным приложением.
- Запустите окружение командой my-app-dock ins.
- ins (instance) — это функция в вашем коде, которая разворачивает контейнер.

## Повседневное использование

- `my-app-dock ins` открывает терминал в запущенном контейнере.
- Либо подключитесь к серверу VSCode
- `my-app-dock clean` удалит все контейнеры принадлежащие `my-app-dock`. Либо `my-app-dock kill [id]` для удаления конкретного

## Разработка
Если приходится часто пересобирать образ, то можно использовать apt-cacher-ng, это кеширующий прокси для пакетов убунты. На хосте
```
sudo apt install apt-cacher-ng
```
Все автоматически поднимется, а x-dock его подхватит во время выполнения следующего apt install внутри контейнера. Посмотреть что закешировано можно в /var/cache/apt-cacher-ng
